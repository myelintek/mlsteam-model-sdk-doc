    "# PyTorch Cifar10 Image Classification\n",
    "\n",
    "💡 This example demonstrates how to use *MLSteam Model SDK* (Archive `v2024.1`) to develop a Cifar10 model for image classification.\n",
    "\n",
    "📘 Cifar10 was introduced by Krizhevsky et al. in [Learning multiple layers of features from tiny images](https://www.cs.toronto.edu/~kriz/learning-features-2009-TR.pdf). The datasets are publicly available in the [author's site](https://www.cs.toronto.edu/~kriz/cifar.html) as well as through many popular ML framework libraries. The PyTorch model training code here originates from a [PyTorch tutorial](https://pytorch.org/tutorials/beginner/blitz/cifar10_tutorial.html)."
   ]

    "## Prepare Files\n",
    "\n",
    "Download the `pytorch-cifar10-multiweights.zip` file and extract it.\n",
    "\n",
    "The extracted files include:\n",
    "\n",
    "```\n",
    "pytorch-cifar10-multiweights/\n",
    "  ├ dataset/\n",
    "  │  └ samples/\n",
    "  │    └ samples.pt  # a torch file containing 4 Cifar10 sample images\n",
    "  ├ hooks/\n",
    "  │  ├ load.py  # default loader hook \n",
    "  │  └ predict.py  # default predictor hook\n",
    "  ├ model/\n",
    "  │  ├ cifar_model-1.pth  # 1st state dict file\n",
    "  │  └ cifar_model-2.pth  # 2nd state dict file\n",
    "  ├ manifest-builtin-hooks.json  # manifest file using built-in hooks\n",
    "  ├ manifest-custom-hooks.json  # manifest file using custom hooks\n",
    "  ├ manifest.json  # manifest file using default hooks\n",
    "  ├ my_hooks.py  # custom hooks\n",
    "  ├ requirements.txt  # Python package dependencies\n",
    "  └ train_multi_weights.py  # Cifar10 model definition & training script\n",
    "```\n",
    "\n",
    "Development typically starts with a model training script, which defines and trains the model from a training dataset. And they add manifest and optionally hook files for model packaging.\n",
    "\n",
    "This example includes more files than necessary for demonstrating 3 ways to definining hooks. It also saves the model in 2 state dict files to show how the SDK deals with multi-weight-file models."
   ]
  },
